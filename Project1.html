How Learning to Code Works

How Learning Anything works

Learning is a journey from being a Novice to becoming an Expert which we can consider to have four destinct phases.

The Stages of Expertise

  1. Ignorance - No knowledge of the subject at all.
  2. Awareness - The point at which you become aware that a subject exists.
  3. Ability - When you can do something but it requires effort and focus to achieve.
  4. Fluency - Knowledge so complete that it comes as easily as breathing.

The journey is an interesting one BUT the time taken from one stage to the next varies significantly depending on the complexity of the subject in question.

  • From Ignorance to Awareness - a journey measured in seconds/minutes
  • From Awareness to Ability - a journey measured in minutes/days.
  • From Ability to Expert - a journey measured in days/weeks/months/years!!!

Remember: Use the online resources available to you (forums, etc.) to ask for help if you are struggling - Don't give up! You CAN do this!'

Avoiding Repetition!

Most Important Rule of Coding
Avoid Repetition!!!

The greatest danger with repetitious coding is that when a change is needed to a routine you may miss some instance of the routine because you haven't relied on a re-useable function to do the routine task.

The Basics of the Web and HTML

The World Wide Web

The World Wide Web is a collection of webpages, images and interactive content (Web applications) connected together by links (Hyperlinks - commonly abbreviated to link) .

Key Components of the Web:

  • The Browser on My Computer
  • The Webserver - located 'out there' - on the Cloud

These two elements communicate via HTTP (Hyper Text Transfer Protocol). The browser sends a request and the webserver returns the requested webpage.

How does the webserver know which page to the browser?

The browser requests a page by clicking on a Hyperlink which contains a reference to a URL (Uniorm Resource Locator) which is unique to each page on the web.

Webpage Overview

Webpages are documents of type HTML (HyperText Markup Language).

HTML comprises four key types of things:

  1. text (what you see)
  2. markup (what it looks like)
  3. references to other documents (video and images))
  4. links to other pages
HTML Fundamentals

The content of an HTML Document is made up of Elements.

All Elements are rectangluar in shape.

HTML <tags> are used to markup these Elements.

These <tags> tell the browser how to handle the individual elements of the webpage.

Most Elements are written with an opening tag, content and a closing tag.
eg. <tag>content</tag>.
In addition, some elements (like images and links) can have (an) attribute(s) which have names and values.
eg. <tag attributeName="value">content</tag>
There are also a number of tags which are referred to as void tags. These tags don't have either content or a closing tag.
Examples:
The image element: <img src="url" alt="text" >
(note: the 'alt' attribute is not compulsory but is important as it is what will be displayed if there is an error/break in the link to the image url. It is also helpful for blind users.)
The Line Break element: <br>

There are two major types or elements:

Inline Elements
Simple in nature and have only content and occupy only the space they need.
Block Elements
Contained within a virtual box and have height and width attributes as well as other potential.
By default they occupy the full width of the webpage.

Creating a Structured Document with HTML

Key Components of a Webpage

A webpage can be likened to a house.

HTML
representing the Structure and Content
CSS
representing the decorative Style
Javascript
representing the interactive elements within, eg. Garage Remote (beyond the scope of this course)

Each of the items above are Languages, they have their own Syntax and Rules.

There are rules to how websites behave.

  • HTML contains the Structure and Content of the webpage
  • HTML uses <tags> to create elements.
  • HTML elements are rectangular in shape (can be considered to be boxes)
  • HTML has a tree-like structure. HTML element content can be simple text, images or other elements (boxes within boxes). Similar to a family tree elements within the tree can have child elements and/or parent elements.
  • CSS (Cascading Style Sheets) contains the style of the webpage

How a webpage is displayed by the browser is controlled by the DOM (Document Object Model).

DOM - a cross-platform and language-independent convention for representing and interacting with objects in HTML.

The DOM uses the HTML elements to build the tree-like structure of the page.

Designing a Webpage - Boxify!

When given a mock-up of a webpage, start looking for boxes.

HTML elements are boxes so ..... find the elements!

  • Obvious first element - the whole page = <body>
  • Keep dividing up the page into boxes (within boxes) until everything on the page is reduced down to the most basic element types - eg. paragraphs or images

Cascading Style Sheets

What is CSS and how does it work?

CSS is code written to control the 'style' of HTML elements.

Styles are applied to Element types. The element types are all the predefined elements of HTML + any user-defined types known as classes.

In order for CSS to know which styles to apply to which elements we need to use a selector. For example: h1 or div
followed by the attributes and values we want applied to those elements contained within curly braces.

h1 {
    background-color : black;
    color: white:
}

How is CSS included on a web page?

There are 3 methods for applying styles with CSS.

  1. In-line in your HTML
    Should only be considered if there is some style attribute that needs to be applied which is exclusive to that particlar instance of that particular class/element. Otherwise it is an extremely inefficient method of coding style information.
    Within the body of your HTML <tag style="background-color: green">content<tag>
  2. Include the main block of your CSS within the <head> element of your HTML.
    Can be considered when the website in question has a very small number of HTML files.
    Add a <style> element within the <head> element. The content part of the <style>content</style> element should be entered as you would if you were creating a separate CSS file.
  3. Link your HTML to a separate CSS file within the <head> element of your HTML
    Write a separate CSS file and link it within the <head> element of your HTML. This is the preferred method when dealing with larger projects as it enables you to put all the standardized style features in one place and apply them across multiple pages giving a uniform look and feel to a whole website. From an efficiency point of view too, the whole look and feel of a website can be updated extremely quickly by making changes on the master styles contained within the website CSS file(s).
    Add a <link> element within your <head> element of your HTML on each page of your website.
    <head>
      <link rel="stylesheet" type="text/css" href="myStyleSheet.css">
    </head>
    Now in the same folder have a file called myStyleSheet.css which contains your CSS.

There are valid reasons for using each of these methods.

Cycle of Development

Code, Test, Refine

  1. Look for natural boxes
  2. Look for repeated styles & semantic elements
  3. Write your HTML
  4. Apply styles (from BIGGEST to smallest)
  5. Fix Things

Keep repeating steps 3 and 4 until the result looks like your page mock.

Use validation tools to check your HTML and CSS to ensure that your code complies to current standards

To verify HTML: http://validator.w3.org/#validate_by_input
To verify CSS: http://www.css-validator.org/#validate_by_input
